# 抽奖号码问题排查指南

## 问题现象
所有用户都显示001号

## 可能原因

### 1. 数据库权限设置不正确

**检查步骤：**
1. 打开云开发控制台
2. 进入"数据库"
3. 找到 `lottery_numbers` 集合
4. 点击"权限设置"

**正确配置：**
- 权限类型：仅创建者可读写
- 这样每个用户只能看到自己创建的记录

**如果权限是"所有用户可读，仅创建者可写"：**
- 所有用户都能看到所有记录
- 查询时会返回第一条记录（001号）
- 需要改为"仅创建者可读写"

### 2. 数据库中有旧数据

**检查步骤：**
1. 打开云开发控制台
2. 进入"数据库"
3. 打开 `lottery_numbers` 集合
4. 查看所有记录

**如果看到多条记录但都是001号：**
说明之前的代码逻辑有问题，创建了多条重复的001号记录

**解决方案：**
1. 删除 `lottery_numbers` 集合中的所有数据
2. 或者删除整个集合，重新创建
3. 确保权限设置为"仅创建者可读写"
4. 重新测试

### 3. 开发工具模拟器问题

**现象：**
- 在开发工具中，不同的"模拟用户"可能共享同一个 openid
- 导致看起来都是001号

**解决方案：**
- 使用真机预览测试
- 或者在开发工具中点击"清除缓存" → "清除数据缓存"
- 然后重新编译

## 完整解决步骤

### 步骤1：清理数据库
```
1. 打开云开发控制台
2. 进入"数据库"
3. 找到 lottery_numbers 集合
4. 点击"删除"按钮，删除所有记录
   或者删除整个集合
```

### 步骤2：重新创建集合（如果删除了集合）
```
1. 点击"添加集合"
2. 集合名称：lottery_numbers
3. 点击"确定"
```

### 步骤3：设置正确的权限
```
1. 点击 lottery_numbers 集合
2. 点击"权限设置"
3. 选择"仅创建者可读写"
4. 点击"保存"
```

### 步骤4：清除小程序缓存
```
1. 在微信开发者工具中
2. 点击"清缓存" → "清除数据缓存"
3. 重新编译小程序
```

### 步骤5：测试
```
1. 使用真机预览（扫码）
2. 或者使用不同的微信账号测试
3. 每个用户应该获得不同的号码：001, 002, 003...
```

## 验证方法

### 在云开发控制台验证：
1. 打开 lottery_numbers 集合
2. 应该看到每条记录都有不同的 _openid
3. number 字段应该是递增的：001, 002, 003...
4. numberValue 字段应该是：1, 2, 3...

### 在小程序中验证：
1. 使用不同的微信账号登录
2. 每个账号应该看到不同的号码
3. 同一个账号多次进入，号码应该保持不变

## 数据库记录示例

正确的数据应该是这样的：

```json
{
  "_id": "xxx1",
  "_openid": "oABC123...",
  "number": "001",
  "numberValue": 1,
  "createTime": "2026-03-01 10:00:00",
  "hasPrize": false
}

{
  "_id": "xxx2",
  "_openid": "oDEF456...",
  "number": "002",
  "numberValue": 2,
  "createTime": "2026-03-01 10:05:00",
  "hasPrize": false
}

{
  "_id": "xxx3",
  "_openid": "oGHI789...",
  "number": "003",
  "numberValue": 3,
  "createTime": "2026-03-01 10:10:00",
  "hasPrize": false
}
```

注意：
- 每条记录的 _openid 应该不同
- number 和 numberValue 应该递增
- 每个用户只能看到自己的记录（权限设置正确的情况下）
