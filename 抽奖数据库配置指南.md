# 抽奖功能数据库配置指南

## 概述

抽奖功能使用云数据库存储号码和中奖信息,不需要云函数。本指南将详细说明如何配置数据库。

## 前置条件

1. 已开通微信小程序云开发
2. 已获取云开发环境ID
3. 已在 `app.js` 中配置环境ID

## 配置步骤

### 第一步: 打开云开发控制台

1. 打开微信开发者工具
2. 点击顶部菜单 "云开发"
3. 进入云开发控制台

### 第二步: 创建数据库集合

#### 1. 创建 lottery_numbers 集合

在数据库页面:
1. 点击 "添加集合"
2. 集合名称输入: `lottery_numbers`
3. 点击 "确定"

#### 2. 创建 lottery_prizes 集合

重复上述步骤:
1. 点击 "添加集合"
2. 集合名称输入: `lottery_prizes`
3. 点击 "确定"

### 第三步: 设置数据库权限 (重要!)

#### lottery_numbers 权限设置

1. 点击 `lottery_numbers` 集合
2. 点击 "权限设置" 标签
3. 选择 "仅创建者可读写"
4. 点击 "保存"

**为什么这样设置?**
- 用户只能看到自己的号码
- 用户可以为自己创建号码记录
- 云数据库会自动添加 `_openid` 字段
- 查询时自动过滤,只返回当前用户的记录

#### lottery_prizes 权限设置

1. 点击 `lottery_prizes` 集合
2. 点击 "权限设置" 标签
3. 选择 "所有用户可读,仅创建者可写"
4. 点击 "保存"

**为什么这样设置?**
- 所有用户都能查询中奖信息
- 防止用户篡改中奖记录
- 管理员通过控制台手动添加中奖记录

### 第四步: 了解数据结构

#### lottery_numbers 集合字段

```json
{
  "_id": "自动生成的记录ID",
  "_openid": "用户唯一标识(自动添加)",
  "number": "001",
  "numberValue": 1,
  "createTime": "2026-02-24T10:30:00.000Z",
  "hasPrize": false
}
```

字段说明:
- `_id`: 记录唯一ID,自动生成
- `_openid`: 用户唯一标识,云数据库自动添加
- `number`: 格式化的号码字符串(如"001")
- `numberValue`: 号码数值,用于排序和计算下一个号码
- `createTime`: 创建时间,使用服务器时间
- `hasPrize`: 是否中奖,默认false

#### lottery_prizes 集合字段

```json
{
  "_id": "自动生成的记录ID",
  "winnerNumber": "088",
  "prizeName": "一等奖",
  "prizeLevel": 1,
  "drawTime": "2026-05-04T14:30:00.000Z"
}
```

字段说明:
- `_id`: 记录唯一ID,自动生成
- `winnerNumber`: 中奖号码(如"088")
- `prizeName`: 奖项名称(一等奖/二等奖/三等奖)
- `prizeLevel`: 奖项等级(1/2/3)
- `drawTime`: 抽奖时间

## 常见问题

### Q: 提示"权限不足"怎么办?

A: 检查数据库权限设置:
- `lottery_numbers`: 必须是 "仅创建者可读写"
- `lottery_prizes`: 必须是 "所有用户可读,仅创建者可写"

### Q: 用户能看到其他人的号码吗?

A: 不能。使用 "仅创建者可读写" 权限后,云数据库会自动过滤,用户只能看到自己的记录。

### Q: 如何手动添加中奖记录?

A: 在云开发控制台:
1. 打开 `lottery_prizes` 集合
2. 点击 "添加记录"
3. 输入中奖信息
4. 点击 "确定"

### Q: 如何查看所有已分配的号码?

A: 在云开发控制台:
1. 打开 `lottery_numbers` 集合
2. 可以看到所有用户的号码
3. 可以按 `numberValue` 排序查看

### Q: 如何重置抽奖数据?

A: 在云开发控制台:
1. 打开对应的集合
2. 选择要删除的记录
3. 点击 "删除"
4. 或者直接删除整个集合重新创建

## 安全建议

1. **不要使用"所有用户可读写"权限**
   - 会导致用户可以修改他人的号码
   - 会导致用户可以篡改中奖记录

2. **定期备份数据**
   - 在云开发控制台导出数据
   - 保存中奖记录作为凭证

3. **监控异常操作**
   - 检查是否有重复号码
   - 检查号码分配是否连续

4. **婚礼前测试**
   - 提前测试完整流程
   - 确保权限配置正确
   - 准备备用方案

## 技术原理

### _openid 自动添加

当使用 "仅创建者可读写" 权限时:
- 云数据库会自动为每条记录添加 `_openid` 字段
- `_openid` 是用户的唯一标识
- 查询时自动添加 `_openid` 过滤条件
- 用户无法看到或修改其他用户的记录

### 号码分配流程

```
1. 用户打开抽奖页面
   ↓
2. 查询 lottery_numbers 集合(自动过滤当前用户)
   ↓
3. 如果有记录 → 显示已有号码
   ↓
4. 如果没有记录 → 分配新号码
   ↓
5. 查询最大 numberValue
   ↓
6. 新号码 = 最大值 + 1
   ↓
7. 保存到数据库(自动添加 _openid)
```

### 中奖查询流程

```
1. 用户下拉刷新
   ↓
2. 查询 lottery_prizes 集合
   ↓
3. 条件: winnerNumber = 用户的号码
   ↓
4. 如果有记录 → 显示中奖信息
   ↓
5. 如果没有记录 → 显示未中奖
```

## 总结

正确配置数据库权限是抽奖功能正常工作的关键:
- ✅ lottery_numbers: 仅创建者可读写
- ✅ lottery_prizes: 所有用户可读,仅创建者可写
- ✅ 不需要云函数
- ✅ 云数据库自动处理用户身份

配置完成后,抽奖功能即可正常使用!
