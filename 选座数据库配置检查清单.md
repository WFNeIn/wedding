# 选座数据库配置检查清单

## 📋 配置检查步骤

请按照以下步骤检查你的数据库配置是否正确：

---

## ✅ 第一步：检查云开发环境

1. 在微信开发者工具中，点击顶部工具栏的 **"云开发"** 按钮
2. 确认云开发控制台能正常打开
3. 确认环境状态显示为 **"正常"**
4. 记录你的环境ID（例如：`wedding-7g2h4m5n`）

**检查结果：**
- [ ] 云开发环境已开通
- [ ] 环境状态正常
- [ ] 环境ID：________________

---

## ✅ 第二步：检查seats集合

1. 在云开发控制台中，点击 **"数据库"** 标签
2. 查看左侧集合列表

**检查结果：**
- [ ] 能看到 `seats` 集合
- [ ] 能看到 `rsvp` 集合（回执功能）
- [ ] 能看到 `blessings` 集合（祝福功能）
- [ ] 能看到 `lottery` 集合（抽奖功能）

**如果没有seats集合，请创建：**
1. 点击 **"添加集合"** 按钮
2. 集合名称输入：`seats`
3. 点击 **"确定"**

---

## ✅ 第三步：检查seats集合权限

1. 点击 `seats` 集合
2. 点击 **"权限设置"** 标签
3. 查看当前权限模式

**检查结果：**
- [ ] 权限模式为：**仅创建者可读写**

**如果权限不正确，请修改：**
1. 在权限设置页面
2. 选择 **"仅创建者可读写"**
3. 点击 **"保存"**

---

## ✅ 第四步：检查seats集合索引（最重要！）

1. 在 `seats` 集合中，点击 **"索引管理"** 标签
2. 查看已创建的索引列表

**检查结果：**

### 索引1：seatId唯一索引（必须！）
- [ ] 索引名称：`seatId_unique` 或类似名称
- [ ] 索引字段：`seatId`
- [ ] 索引类型：**唯一索引** ⚠️
- [ ] 排序方式：升序

### 索引2：_openid索引（推荐）
- [ ] 索引名称：`openid_index` 或类似名称
- [ ] 索引字段：`_openid`
- [ ] 索引类型：普通索引
- [ ] 排序方式：升序

### 索引3：guestType索引（推荐）
- [ ] 索引名称：`guestType_index` 或类似名称
- [ ] 索引字段：`guestType`
- [ ] 索引类型：普通索引
- [ ] 排序方式：升序

**如果缺少索引，请创建：**

#### 创建seatId唯一索引（必须！）
1. 点击 **"添加索引"** 按钮
2. 填写配置：
   - 索引名称：`seatId_unique`
   - 索引字段：`seatId`
   - 索引类型：选择 **"唯一索引"** ⚠️
   - 排序方式：升序
3. 点击 **"确定"**

#### 创建_openid索引（推荐）
1. 点击 **"添加索引"** 按钮
2. 填写配置：
   - 索引名称：`openid_index`
   - 索引字段：`_openid`
   - 索引类型：普通索引
   - 排序方式：升序
3. 点击 **"确定"**

#### 创建guestType索引（推荐）
1. 点击 **"添加索引"** 按钮
2. 填写配置：
   - 索引名称：`guestType_index`
   - 索引字段：`guestType`
   - 索引类型：普通索引
   - 排序方式：升序
3. 点击 **"确定"**

---

## ⚠️ 常见问题排查

### 问题1：提示"云函数不存在"
**原因**：云函数未部署
**解决**：按照第五步部署所有云函数

### 问题2：选座时提示"座位已被占用"但实际没人选
**原因**：可能有测试数据残留
**解决**：在云开发控制台 → 数据库 → seats集合 → 删除所有记录

### 问题3：多人同时选同一座位都成功了
**原因**：seatId唯一索引未创建
**解决**：必须创建seatId唯一索引！这是防止并发冲突的关键

### 问题4：查询速度很慢
**原因**：缺少索引
**解决**：创建_openid和guestType索引

### 问题5：提示"权限不足"
**原因**：集合权限配置错误
**解决**：将seats集合权限设置为"仅创建者可读写"
