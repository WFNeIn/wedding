# 天气查询云函数

## 功能说明

根据经纬度查询当前位置的实时天气信息，用于在首页婚宴地点卡片上显示天气。

## 主要特性

1. **智能缓存机制**：每天只请求一次API，大幅减少API调用次数
2. **多级降级方案**：API失败时依次尝试昨天缓存、模拟数据
3. **参数校验**：确保输入参数的有效性
4. **并行请求**：同时获取实时天气和预报数据，提升性能
5. **自动清理**：自动删除7天前的旧缓存数据
6. **超时控制**：API请求设置10秒超时，避免长时间等待

## 优化点

### 1. 缓存策略优化
- 使用 upsert 模式更新缓存，避免重复数据
- 缓存查询失败不影响主流程
- 自动清理过期缓存，节省存储空间

### 2. 性能优化
- 并行请求实时天气和预报数据（Promise.all）
- 添加请求超时控制（10秒）
- 经纬度保留4位小数，减少缓存key数量

### 3. 容错优化
- 参数校验，提前返回错误
- API失败时尝试使用昨天的缓存
- 多级降级方案确保始终有数据返回
- 所有异常都有日志记录

### 4. 代码优化
- 提取函数，提高代码可读性
- 移除未使用的参数（context）
- 添加详细的错误信息和日志

## 部署步骤

### 1. 创建数据库集合

在云开发控制台创建 `weather_cache` 集合，权限设置为"仅创建者可读写"。

### 2. 安装依赖

在微信开发者工具中：
1. 右键点击 `cloudfunctions/getWeather` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成

### 3. 配置和风天气API（可选）

如果需要获取真实天气数据：

1. 访问 https://dev.qweather.com/ 注册账号
2. 创建应用，选择"免费订阅"
3. 获取 API Key
4. 在 `index.js` 中将 `YOUR_API_KEY_HERE` 替换为实际的 API Key

```javascript
const apiKey = 'YOUR_API_KEY_HERE'; // 替换为实际的API key
```

### 4. 测试云函数

在微信开发者工具的云开发控制台中：
1. 进入"云函数"页面
2. 找到 `getWeather` 函数
3. 点击"测试"
4. 输入测试参数：
```json
{
  "latitude": 30.826285,
  "longitude": 114.811298
}
```
5. 查看返回结果

## 返回数据格式

```json
{
  "success": true,
  "data": {
    "weather": "晴",
    "temp": "25",
    "tempRange": "18~28",
    "windDir": "东南风",
    "windScale": "3级",
    "humidity": "65",
    "text": "晴朗",
    "updateTime": "2024-01-01T12:00:00+08:00"
  },
  "message": "缓存数据",
  "cached": true
}
```

## 缓存机制说明

1. **首次请求**：调用API获取天气，保存到数据库
2. **当天再次请求**：直接返回缓存数据，不调用API
3. **第二天请求**：缓存过期，重新调用API
4. **API失败**：尝试返回昨天的缓存数据
5. **无缓存**：返回模拟数据

## 数据库结构

`weather_cache` 集合：

```javascript
{
  _id: "自动生成",
  cacheKey: "weather_30.8263_114.8113",
  date: "2024-01-01",
  weatherData: {
    weather: "晴",
    temp: "25",
    // ... 其他天气数据
  },
  createTime: "服务器时间",
  updateTime: "服务器时间"
}
```

## 注意事项

1. 和风天气免费版有调用次数限制（每天1000次）
2. 缓存机制确保每天每个位置只调用一次API
3. 自动清理7天前的旧缓存，节省存储空间
4. 建议在生产环境配置真实API Key
5. 如果API调用失败，会自动降级到模拟数据
