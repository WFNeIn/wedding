# 祝福留言匿名功能说明

## 功能概述

祝福留言板采用匿名显示机制,保护用户隐私的同时,在数据库中记录真实信息供管理员查看。

## 工作原理

### 前端显示 (用户看到的)

所有留言都显示为自动生成的匿名昵称,例如:
- 热心的宾客
- 友善的朋友
- 可爱的来宾
- 温暖的好友
- 善良的伙伴
- ...

昵称生成规则:
- 使用10个前缀词: 热心、友善、可爱、温暖、善良、真诚、快乐、幸福、美好、甜蜜
- 使用5个后缀词: 宾客、朋友、来宾、好友、伙伴
- 按留言顺序自动组合生成
- 如果超过50个组合,会添加数字后缀(如: 热心的宾客2)

### 数据库存储 (管理员看到的)

每条留言在数据库中包含以下字段:

```json
{
  "_id": "记录ID",
  "_openid": "用户的微信openid(自动添加)",
  "realNickname": "用户的真实微信昵称",
  "nickname": "匿名用户(占位字段)",
  "content": "祝福内容",
  "createTime": "创建时间"
}
```

关键字段说明:
- `_openid`: 云数据库自动添加,用户的唯一标识
- `realNickname`: 用户授权获取的真实微信昵称
- `nickname`: 占位字段,前端不使用此字段
- `content`: 祝福内容
- `createTime`: 服务器时间

## 数据查看方式

### 方法1: 云开发控制台

1. 登录微信开发者工具
2. 点击"云开发"按钮
3. 进入"数据库"
4. 点击 `blessings` 集合
5. 查看所有留言记录

可以看到:
- 每条留言的真实昵称 (`realNickname`)
- 用户的openid (`_openid`)
- 留言内容和时间

### 方法2: 导出数据

1. 在云开发控制台的数据库页面
2. 点击 `blessings` 集合
3. 点击"导出"按钮
4. 选择导出格式(JSON或CSV)
5. 下载到本地查看

导出的数据包含所有字段,可以用Excel打开查看。

## 隐私保护说明

### 用户隐私保护

1. **前端匿名显示**
   - 所有用户看到的都是匿名昵称
   - 无法通过小程序界面看到真实身份
   - 保护用户隐私,鼓励真实表达

2. **授权机制**
   - 首次发送祝福时,会请求用户授权获取昵称
   - 用户可以选择拒绝授权
   - 拒绝授权时,使用"访客"作为昵称

3. **数据安全**
   - 真实信息只存储在云数据库
   - 只有小程序管理员可以查看
   - 云数据库有完善的安全机制

### 管理员权限

只有以下人员可以查看真实信息:
- 小程序管理员
- 云开发环境的管理员
- 有数据库访问权限的开发者

## 使用场景

### 为什么要匿名显示?

1. **保护隐私**
   - 避免用户因为身份暴露而不敢真实表达
   - 减少社交压力

2. **鼓励参与**
   - 匿名环境下,用户更愿意留言
   - 提高互动率

3. **避免攀比**
   - 不会因为看到某些人的名字而产生压力
   - 营造平等友好的氛围

### 为什么要记录真实信息?

1. **数据统计**
   - 了解哪些宾客留言了
   - 统计参与度

2. **内容管理**
   - 如有不当内容,可以追溯来源
   - 必要时可以联系用户

3. **纪念意义**
   - 婚礼后可以导出数据
   - 制作纪念册时可以看到真实姓名

## 技术实现

### 匿名昵称生成算法

```javascript
generateAnonymousName(index) {
  const prefixes = ['热心', '友善', '可爱', '温暖', '善良', '真诚', '快乐', '幸福', '美好', '甜蜜'];
  const suffixes = ['宾客', '朋友', '来宾', '好友', '伙伴'];
  
  const prefixIndex = index % prefixes.length;
  const suffixIndex = Math.floor(index / prefixes.length) % suffixes.length;
  const number = Math.floor(index / (prefixes.length * suffixes.length)) + 1;
  
  if (number === 1) {
    return `${prefixes[prefixIndex]}的${suffixes[suffixIndex]}`;
  } else {
    return `${prefixes[prefixIndex]}的${suffixes[suffixIndex]}${number}`;
  }
}
```

特点:
- 基于留言顺序生成
- 确保每个昵称在当前列表中唯一
- 可以生成无限多个不重复的昵称

### 数据流程

```
用户发送祝福
    ↓
请求获取用户信息(昵称)
    ↓
用户授权/拒绝
    ↓
保存到数据库:
  - realNickname: 真实昵称
  - _openid: 自动添加
  - content: 祝福内容
    ↓
刷新留言列表
    ↓
前端生成匿名昵称显示
```

## 常见问题

### Q1: 用户能看到自己的真实昵称吗?
A: 不能。所有用户(包括发送者自己)看到的都是匿名昵称。

### Q2: 如何修改匿名昵称的词库?
A: 编辑 `pages/blessings/blessings.js` 中的 `generateAnonymousName` 方法,修改 `prefixes` 和 `suffixes` 数组。

### Q3: 可以让用户选择是否匿名吗?
A: 可以,但需要修改代码添加选项。当前设计是全部匿名,简化用户操作。

### Q4: 数据库中的 nickname 字段有什么用?
A: 这是一个占位字段,保持数据结构的兼容性。实际显示时不使用此字段。

### Q5: 如果用户拒绝授权怎么办?
A: 会使用"访客"作为 realNickname 保存到数据库,前端仍然显示匿名昵称。

### Q6: 匿名昵称会重复吗?
A: 在当前显示的列表中不会重复。如果留言数量超过50条,会添加数字后缀区分。

### Q7: 可以导出带真实昵称的留言吗?
A: 可以。在云开发控制台导出数据,包含所有字段,包括 realNickname。

```javascript
  // 其余代码保持不变
  ...

```

如果需要记录更多信息,可以在 `submitBlessing` 方法中添加:

```javascript
db.collection('blessings').add({
  data: {

    nickname: '匿名用户',

    userGender: gender,    // 用户性别
    userCity: city         // 用户城市
  }

```
