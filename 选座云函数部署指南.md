# 选座云函数部署指南

## 概述

选座功能需要部署4个云函数：
1. **selectSeat** - 选择座位
2. **getSeats** - 获取所有座位状态
3. **getMySeat** - 获取我的座位
4. **getAllSeats** - 管理员查看所有座位（含统计）

## 部署前准备

### 1. 确认云开发环境已开通

在微信开发者工具中：
- 点击"云开发"按钮
- 确认环境已创建且状态正常
- 记录环境ID（格式如：wedding-7g2h4m5n）

### 2. 确认数据库已配置

必须先完成数据库配置，包括：
- ✅ 创建seats集合
- ✅ 创建seatId唯一索引（重要！）
- ✅ 创建_openid和guestType普通索引
- ✅ 设置集合权限为"仅创建者可读写"

详见：`选座数据库配置指南.md`

## 云函数部署步骤

### 方法1: 使用微信开发者工具部署（推荐）

#### 1. 部署selectSeat云函数

1. 在微信开发者工具中，找到 `cloudfunctions/selectSeat` 文件夹
2. 右键点击 `selectSeat` 文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待上传完成（会自动安装wx-server-sdk依赖）
5. 查看控制台确认部署成功

#### 2. 部署getSeats云函数

1. 右键点击 `cloudfunctions/getSeats` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待上传完成

#### 3. 部署getMySeat云函数

1. 右键点击 `cloudfunctions/getMySeat` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待上传完成

#### 4. 部署getAllSeats云函数

1. 右键点击 `cloudfunctions/getAllSeats` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待上传完成

### 方法2: 使用命令行部署

如果安装了云开发CLI工具，可以使用命令行部署：

```bash
# 进入云函数目录
cd cloudfunctions

# 部署selectSeat
cd selectSeat
npm install
tcb fn deploy selectSeat

# 部署getSeats
cd ../getSeats
npm install
tcb fn deploy getSeats

# 部署getMySeat
cd ../getMySeat
npm install
tcb fn deploy getMySeat

# 部署getAllSeats
cd ../getAllSeats
npm install
tcb fn deploy getAllSeats
```

## 常见问题

### Q1: 上传云函数时提示"环境不存在"？

A: 检查以下几点：
1. 确认云开发环境已开通
2. 在app.js中配置了正确的环境ID
3. 在微信开发者工具中选择了正确的云开发环境

### Q2: 云函数部署成功但调用失败？

A: 可能的原因：
1. 数据库集合未创建
2. 数据库权限配置错误
3. 云函数代码有语法错误（查看云函数日志）

### Q3: 提示"唯一索引冲突"？

A: 这是正常的！说明座位已被占用。确保seatId字段已创建唯一索引。

### Q4: 如何查看云函数日志？

A: 在云开发控制台：
1. 点击"云函数"
2. 点击具体的云函数
3. 点击"日志"标签
4. 可以看到调用记录和错误信息

### Q5: 云函数调用超时？

A: 检查：
1. 网络连接是否正常
2. 数据库查询是否有索引（提高查询速度）
3. 云函数代码是否有死循环

### Q6: 如何更新云函数？

A: 修改代码后，重新右键上传即可。建议选择"上传并部署：云端安装依赖"。

## 性能优化建议

### 1. 数据库索引

确保创建了所有必需的索引：
- seatId唯一索引（必须！）
- _openid普通索引
- guestType普通索引

### 2. 缓存策略

可以在小程序端缓存座位数据，减少云函数调用：
```javascript
// 缓存5分钟
const CACHE_TIME = 5 * 60 * 1000;
```

### 3. 批量操作

如果需要批量操作座位，可以创建专门的云函数，避免多次调用。

## 安全建议

### 1. 权限控制

- seats集合权限设置为"仅创建者可读写"
- 确保用户只能操作自己的座位记录

### 2. 参数验证

云函数已包含完整的参数验证，确保：
- row范围：1-4
- col范围：1-5
- side值：left或right
- guestType值：有效的类型ID

### 3. 并发控制

使用数据库唯一索引确保座位唯一性，无需额外的锁机制。
