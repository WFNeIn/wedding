# 部署指南

本文档说明如何将婚宴请帖小程序部署到微信平台。

## 前置准备

### 1. 账号准备
- ✅ 已注册微信小程序账号
- ✅ 已获取小程序 AppID
- ✅ 已完成小程序实名认证（云开发需要）

### 2. 开发工具
- ✅ 已安装微信开发者工具（最新版本）
- ✅ 已登录微信开发者工具

### 3. 资源准备
- ✅ 3-4张婚纱照（每张 < 500KB）
- ✅ 8个Tab图标（可选，或使用纯文字导航）

## 部署步骤

### 第一步：配置项目

#### 1.1 修改 AppID

编辑 `project.config.json`：

```json
{
  "appid": "wx1234567890abcdef",  // 替换为你的AppID
  ...
}
```

或在微信开发者工具导入项目时直接输入 AppID。

#### 1.2 修改婚礼信息

编辑 `app.js` 中的 `globalData.weddingInfo`：

```javascript
weddingInfo: {
  couple: {
    groom: '新郎姓名',  // 修改新郎姓名
    bride: '新娘姓名'   // 修改新娘姓名
  },
  date: '2025-05-04',   // 修改婚礼日期
  time: '12:00',        // 修改婚礼时间
  venue: {
    name: '酒店名称',   // 修改酒店名称
    address: '详细地址', // 修改详细地址
    latitude: 30.826285,  // 修改纬度
    longitude: 114.811298 // 修改经度
  },
  schedule: [
    { time: '11:00', event: '迎宾' },
    { time: '12:00', event: '仪式' },
    { time: '12:30', event: '午宴' }
  ],
  contact: {
    name: '新郎',
    phone: '13900000000'  // 修改联系电话
  }
}
```

**获取地图坐标**：
访问 [腾讯位置服务](https://lbs.qq.com/getPoint/) 获取酒店的经纬度坐标。

#### 1.3 准备图片资源

将婚纱照放入 `images/` 目录，命名为：
- `20260223-183518.jpg`
- `20260223-183532.jpg`
- `20260223-183537.jpg`
- `20260223-183542.jpg`

**图片要求**：
- 格式：JPG 或 PNG
- 尺寸：建议 750px × 1000px
- 大小：每张 < 500KB

**压缩图片**：
- 在线工具：[TinyPNG](https://tinypng.com/)
- 或使用图片编辑软件导出时选择"Web优化"

### 第二步：开通云开发

#### 2.1 开通云开发服务

1. 在微信开发者工具中点击"云开发"按钮
2. 点击"开通"，同意服务协议
3. 创建云开发环境：
   - 环境名称：`wedding-prod`（或自定义）
   - 选择基础版（免费）
4. 等待环境创建完成（约1-2分钟）

#### 2.2 获取并配置环境ID

1. 创建完成后，复制环境ID（如：`wedding-7g2h4m5n`）
2. 编辑 `app.js`，修改云开发配置：

```javascript
wx.cloud.init({
  env: 'wedding-7g2h4m5n',  // 替换为你的环境ID
  traceUser: true
});
```

#### 2.3 创建数据库集合

1. 在云开发控制台，点击"数据库"
2. 点击"添加集合"，创建 `rsvp` 集合
3. 再次点击"添加集合"，创建 `blessings` 集合
4. 权限设置：选择"所有用户可读写"（开发阶段）

详细步骤请参考：`cloudfunctions/DATABASE_SETUP.md`

#### 2.4 部署云函数（可选）

如果要使用云函数方式（推荐生产环境）：

1. 右键点击 `cloudfunctions/submitRSVP` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成
4. 重复以上步骤部署 `addBlessing` 云函数

详细说明请参考：`cloudfunctions/README.md`

**注意**：当前代码使用直接数据库方式，云函数是可选的。

### 第三步：测试功能

#### 3.1 本地测试

1. 在微信开发者工具中点击"编译"
2. 测试各个页面功能：
   - ✅ 首页：轮播图、倒计时显示
   - ✅ 详情页：流程、地图导航、拨号
   - ✅ 回执页：表单填写、提交
   - ✅ 留言板：查看留言、发送留言
3. 测试底部导航栏切换

#### 3.2 真机测试

1. 点击工具栏的"预览"按钮
2. 使用微信扫描二维码
3. 在真机上测试所有功能
4. 特别测试：
   - 地图导航是否正常
   - 拨打电话是否正常
   - 数据提交是否成功

#### 3.3 检查数据库

1. 在云开发控制台查看"数据库"
2. 点击 `rsvp` 集合，查看是否有测试数据
3. 点击 `blessings` 集合，查看是否有测试留言

### 第四步：上传代码

#### 4.1 上传小程序代码

1. 点击工具栏的"上传"按钮
2. 填写版本号（如：1.0.0）
3. 填写项目备注（如：初始版本）
4. 点击"上传"

#### 4.2 提交审核

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"版本管理"
3. 找到刚上传的版本，点击"提交审核"
4. 填写审核信息：
   - 功能页面：选择所有页面
   - 测试账号：提供测试账号（如需要）
   - 补充说明：简要说明小程序功能
5. 提交审核

#### 4.3 等待审核

- 审核时间：通常1-7个工作日
- 审核结果：会通过公众平台消息通知
- 如果审核不通过，根据反馈修改后重新提交

#### 4.4 发布上线

1. 审核通过后，在"版本管理"中点击"发布"
2. 确认发布
3. 发布成功后，用户即可搜索到你的小程序

## 常见问题

### Q1: 编译报错"云开发未初始化"
**A**: 检查 `app.js` 中的环境ID是否正确配置。

### Q2: 图片不显示
**A**: 检查图片路径和文件名是否正确，图片是否已放入 `images/` 目录。

### Q3: 数据提交失败
**A**: 
1. 检查云开发是否已开通
2. 检查数据库集合是否已创建
3. 检查数据库权限设置
4. 查看控制台错误日志

### Q4: 地图导航无法打开
**A**: 
1. 检查经纬度坐标是否正确
2. 在真机上测试（开发者工具可能无法完全模拟）
3. 检查用户是否授予位置权限

### Q5: Tab图标不显示
**A**: 
1. 检查图标文件是否存在
2. 检查图标路径是否正确
3. 或者使用纯文字导航（删除 iconPath 配置）

### Q6: 审核不通过
**A**: 常见原因：
- 缺少隐私政策
- 功能描述不清晰
- 测试账号无法登录
- 根据审核反馈修改后重新提交

## 优化建议

### 性能优化
1. **图片压缩**：确保所有图片都已压缩
2. **代码分包**：如果代码超过2MB，考虑分包加载
3. **懒加载**：留言列表使用分页加载

### 安全优化
1. **使用云函数**：生产环境建议使用云函数处理数据
2. **数据库权限**：设置更严格的权限规则
3. **内容审核**：接入微信内容安全API

### 用户体验优化
1. **加载提示**：所有异步操作都显示加载提示
2. **错误提示**：提供清晰的错误提示信息
3. **空状态**：列表为空时显示友好提示
