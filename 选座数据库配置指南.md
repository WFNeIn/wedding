# 选座数据库配置指南

## 数据库集合创建

### 1. 打开云开发控制台

在微信开发者工具中：
1. 点击顶部工具栏的"云开发"按钮
2. 进入云开发控制台
3. 选择"数据库"标签

### 2. 创建seats集合

1. 点击"添加集合"按钮
2. 集合名称输入：`seats`
3. 点击"确定"创建

### 3. 配置集合权限

1. 点击seats集合
2. 点击"权限设置"标签
3. 选择权限模式：**仅创建者可读写**
   - 这样可以确保每个用户只能读写自己的座位记录
   - 防止用户修改他人的座位信息

### 4. 创建索引

#### 4.1 创建seatId唯一索引（重要！）

1. 在seats集合中，点击"索引管理"标签
2. 点击"添加索引"
3. 配置如下：
   - 索引名称：`seatId_unique`
   - 索引字段：`seatId`
   - 索引类型：**唯一索引**
   - 排序方式：升序
4. 点击"确定"

**作用**：确保同一座位不能被多个用户同时占用，这是并发控制的关键！

#### 4.2 创建_openid索引

1. 点击"添加索引"
2. 配置如下：
   - 索引名称：`openid_index`
   - 索引字段：`_openid`
   - 索引类型：普通索引
   - 排序方式：升序
3. 点击"确定"

**作用**：加速查询用户自己的座位记录

#### 4.3 创建guestType索引

1. 点击"添加索引"
2. 配置如下：
   - 索引名称：`guestType_index`
   - 索引字段：`guestType`
   - 索引类型：普通索引
   - 排序方式：升序
3. 点击"确定"

**作用**：加速按宾客类型统计查询

## 数据结构说明

### seats集合文档结构

```javascript
{
  _id: "自动生成的文档ID",
  _openid: "用户的微信openid（自动添加）",
  seatId: "left_1_1",           // 座位唯一标识
  row: 1,                        // 排号 (1-4)
  col: 1,                        // 列号 (1-5)
  side: "left",                  // 左侧'left'或右侧'right'
  guestType: "classmate",        // 宾客类型ID
  guestTypeName: "同学同事",      // 宾客类型名称
  createTime: Date,              // 选座时间
  updateTime: Date               // 更新时间
}
```

### 宾客类型枚举值

- `classmate`: 同学同事
- `father_colleague`: 父亲的同事
- `mother_colleague`: 母亲的同事
- `parent_friend`: 父母的亲友

## 验证配置

### 1. 检查集合是否创建成功

在云开发控制台的数据库页面，应该能看到`seats`集合

### 2. 检查索引是否创建成功

点击seats集合 → 索引管理，应该能看到3个索引：
- ✅ seatId_unique（唯一索引）
- ✅ openid_index（普通索引）
- ✅ guestType_index（普通索引）

### 3. 检查权限设置

点击seats集合 → 权限设置，应该显示：**仅创建者可读写**

## 常见问题

### Q1: 为什么seatId必须是唯一索引？

A: 唯一索引可以防止多个用户同时选择同一座位。当两个用户同时提交选择同一座位时，数据库会自动拒绝第二个请求，确保座位唯一性。

### Q2: 如果忘记创建唯一索引会怎样？

A: 可能会出现多个用户占用同一座位的情况，导致座位冲突。必须在部署前创建唯一索引！

### Q3: 如何清空所有座位数据？

A: 在云开发控制台 → 数据库 → seats集合 → 记录管理，可以批量删除记录。注意：这个操作不可恢复！

### Q4: 如何查看当前有多少座位被占用？

A: 在云开发控制台 → 数据库 → seats集合 → 记录管理，可以看到记录总数，即已占用的座位数量。
