# 选座区域限制修复说明

## 问题描述

用户反馈区域限制有问题。经检查发现云函数 `selectSeat` 中的代码与前端不一致。

## 发现的问题

### 1. 宾客类型不匹配
**云函数（旧）**：
```javascript
const GUEST_TYPE_MAP = {
  'classmate': '同学同事',
  'father_colleague': '父亲的同事',
  'mother_colleague': '母亲的同事',
  'parent_friend': '父母的亲友'
};
```

**前端（新）**：
```javascript
const GUEST_TYPES = [
  { id: 'classmate', name: '同学' },
  { id: 'bride_family', name: '女方亲友' },
  { id: 'groom_family', name: '男方亲友' }
];
```

### 2. 桌号范围错误
**云函数（旧）**：验证桌号1-10
**实际需求**：左侧1-15桌，右侧16-21桌

### 3. 缺少区域限制验证
云函数没有验证用户选择的座位是否在其宾客类型对应的区域内。

## 修复内容

### 1. 更新宾客类型映射
```javascript
const GUEST_TYPE_MAP = {
  'classmate': '同学',
  'bride_family': '女方亲友',
  'groom_family': '男方亲友'
};
```

### 2. 修正桌号验证
```javascript
function validateParams(tableNum, seatNum, side, guestType) {
  // 左侧15桌（1-15），右侧6桌（16-21）
  if (side === 'left') {
    if (!tableNum || tableNum < 1 || tableNum > 15) {
      return { valid: false, message: '左侧桌号必须在1-15之间' };
    }
  } else if (side === 'right') {
    if (!tableNum || tableNum < 16 || tableNum > 21) {
      return { valid: false, message: '右侧桌号必须在16-21之间' };
    }
  }
  // ...
}
```

### 3. 添加区域限制验证

新增两个函数：

**获取座位区域类型**：
```javascript
function getSeatAreaType(tableNum) {
  if (tableNum <= 2) {
    return 'classmate';        // 1-2桌：同学区
  } else if (tableNum <= 8) {
    return 'bride_family';     // 3-8桌：女方亲友区
  } else {
    return 'groom_family';     // 9-21桌：男方亲友区
  }
}
```

**验证区域限制**：
```javascript
function validateAreaRestriction(tableNum, guestType) {
  const seatAreaType = getSeatAreaType(tableNum);
  
  if (seatAreaType !== guestType) {
    const areaNames = {
      'classmate': '同学区（1-2桌）',
      'bride_family': '女方亲友区（3-8桌）',
      'groom_family': '男方亲友区（9-21桌）'
    };
    
    return {
      valid: false,
      message: `您选择的是"${GUEST_TYPE_MAP[guestType]}"类型，只能在${areaNames[guestType]}选座`
    };
  }
  
  return { valid: true };
}
```

### 4. 在主流程中调用验证
```javascript
// 3. 验证区域限制
const areaValidation = validateAreaRestriction(tableNum, guestType);
if (!areaValidation.valid) {
  return {
    success: false,
    message: areaValidation.message
  };
}
```

### 5. 修正备用桌检查
从检查桌号10改为检查桌号21：
```javascript
// 2. 检查是否是备用桌（21号桌）
if (tableNum === 21) {
  return {
    success: false,
    message: '该桌为备用桌，暂不开放选座'
  };
}
```

## 区域划分规则

| 区域 | 桌号范围 | 宾客类型 | 颜色 |
|------|---------|---------|------|
| 同学区 | 1-2桌 | classmate | 红色 #FF6B6B |
| 女方亲友区 | 3-8桌 | bride_family | 粉色 #FFB6C1 |
| 男方亲友区 | 9-21桌 | groom_family | 蓝色 #87CEEB |
| 备用桌 | 21桌 | - | 灰色（不可选） |

## 验证流程

用户选座时的完整验证流程：

1. ✅ 验证参数格式（桌号、座位号、左右侧、宾客类型）
2. ✅ 检查是否是备用桌（21号桌）
3. ✅ **验证区域限制**（新增）
4. ✅ 检查用户是否已选座
5. ✅ 尝试插入座位记录（并发控制）
