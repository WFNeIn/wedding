# 获取正确的云开发环境ID

## 🚨 问题说明

错误代码 `-601034` 表示：当前配置的环境ID `app-env-wf-invitation-3a8672c704` 不属于你的小程序，或者该环境未开通。

---

## ✅ 解决步骤（3分钟）

### 步骤1：打开云开发控制台

1. 在微信开发者工具顶部，点击 **"云开发"** 按钮（☁️ 图标）

2. 会出现以下两种情况之一：

#### 情况A：提示"开通云开发"
如果看到开通提示：
- 点击 **"开通"** 按钮
- 同意服务协议
- 创建环境：
  - 环境名称：`wedding`（或任意英文名）
  - 选择 **"基础版"**（免费）
- 等待创建完成（1-2分钟）

#### 情况B：直接进入云开发控制台
如果直接进入控制台，说明已经开通过云开发。

---

### 步骤2：查看并复制环境ID

在云开发控制台：

1. **查看顶部的环境信息**
   - 在控制台顶部，你会看到当前环境的名称和ID
   - 环境ID格式类似：
     - `cloud1-xxx`
     - `wedding-xxx`
     - `prod-xxx`

2. **复制环境ID**
   - 点击环境ID旁边的复制按钮
   - 或者手动选中并复制

**示例**：
```
环境名称：wedding
环境ID：cloud1-7g2h4m5n  ← 复制这个
```

---

### 步骤3：修改 app.js 配置

1. 打开项目中的 `app.js` 文件

2. 找到这一行：
```javascript
env: 'app-env-wf-invitation-3a8672c704',
```

3. 替换为你复制的环境ID：
```javascript
env: 'cloud1-7g2h4m5n',  // 改成你的环境ID
```

4. **保存文件**（Ctrl+S 或 Cmd+S）

---

### 步骤4：创建数据库集合

在云开发控制台：

1. 点击左侧菜单 **"数据库"**

2. 点击 **"添加集合"**
   - 集合名称：`blessings`
   - 点击"确定"

3. 点击 `blessings` 集合名称

4. 点击 **"权限设置"** 标签

5. 选择 **"所有用户可读写"**

6. 点击 **"保存"**

7. 重复上述步骤，创建 `rsvp` 集合

---

### 步骤5：重新编译测试

1. 在微信开发者工具中点击 **"编译"** 按钮

2. 查看控制台（Console），应该看到：
```
小程序启动
开始初始化云开发...
云开发初始化成功
环境ID: cloud1-7g2h4m5n
```

3. 切换到"祝福"页面，输入留言并发送

4. 如果成功，会提示"发送成功"

---

## 🔍 如何确认环境ID是否正确？

### 方法1：查看控制台顶部
在云开发控制台顶部，环境ID会显示在环境名称旁边。

### 方法2：查看设置页面
1. 在云开发控制台，点击左侧"设置"
2. 在"环境设置"中可以看到完整的环境ID

### 方法3：使用命令行（高级）
在控制台（Console）输入：
```javascript
wx.cloud.database().collection('blessings').get().then(res => {
  console.log('环境正确，测试成功');
}).catch(err => {
  console.error('环境错误:', err);
});
```

## ❓ 常见问题

### Q1: 我有多个环境，应该用哪个？
**A**: 
- 如果只有一个环境，就用那个
- 如果有多个，建议使用名称包含 "prod"、"production" 或 "wedding" 的环境
- 或者新建一个专门用于婚礼小程序的环境

### Q2: 环境ID在哪里显示？
**A**: 
在云开发控制台顶部，格式如下：
```
[环境名称] wedding  |  [环境ID] cloud1-7g2h4m5n
```

### Q3: 我没有云开发按钮怎么办？
**A**: 
- 确保使用的是真实的 AppID（不是测试号）
- 测试号不支持云开发功能
- 需要先注册小程序账号

### Q4: 提示"该小程序未开通云开发"
**A**: 
- 点击"开通"按钮
- 按照提示创建环境
- 选择"基础版"（免费）

### Q5: 创建环境失败
**A**: 
- 检查网络连接
- 确保微信账号已实名认证
- 尝试更换环境名称（只用英文和数字）
- 刷新页面重试
