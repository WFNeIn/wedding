# 天气功能部署检查清单

## 部署前准备

- [ ] 确认已开通微信云开发
- [ ] 确认云开发环境ID已配置在 `app.js` 中
- [ ] 确认网络连接正常

## 步骤1：创建数据库集合

- [ ] 打开云开发控制台
- [ ] 进入"数据库"页面
- [ ] 创建集合：`weather_cache`
- [ ] 设置权限：仅创建者可读写
- [ ] （可选）添加索引：`cacheKey_date` 和 `date`

详细步骤见：[天气缓存数据库配置指南.md](./天气缓存数据库配置指南.md)

## 步骤2：部署云函数

- [ ] 在微信开发者工具中找到 `cloudfunctions/getWeather` 文件夹
- [ ] 右键点击该文件夹
- [ ] 选择"上传并部署：云端安装依赖"
- [ ] 等待部署完成（查看控制台输出）
- [ ] 确认部署成功（云开发控制台 → 云函数 → 看到 getWeather）

## 步骤3：测试云函数

- [ ] 在云开发控制台，进入"云函数"页面
- [ ] 找到 `getWeather` 函数
- [ ] 点击"测试"按钮
- [ ] 输入测试参数：
```json
{
  "latitude": 30.826285,
  "longitude": 114.811298
}
```
- [ ] 点击"运行测试"
- [ ] 查看返回结果，确认 `success: true`
- [ ] 查看云函数日志，确认无错误

## 步骤4：测试小程序

- [ ] 在微信开发者工具中点击"编译"
- [ ] 打开"控制台"
- [ ] 查看日志输出：
  - [ ] 看到"开始加载天气，经纬度: ..."
  - [ ] 看到"天气云函数返回: ..."
  - [ ] 看到"天气数据: ..." 或 "使用模拟天气数据"
  - [ ] 看到"weatherData已更新: ..."
- [ ] 在首页婚宴地点卡片上看到天气信息显示

## 步骤5：验证缓存功能

- [ ] 第一次加载页面，查看云函数日志
  - [ ] 应该看到"缓存未命中，调用API获取天气"
  - [ ] 应该看到"天气缓存已创建"
- [ ] 刷新页面（第二次加载）
  - [ ] 应该看到"使用缓存数据"
  - [ ] 不应该调用天气API
- [ ] 在云开发控制台 → 数据库 → weather_cache
  - [ ] 应该看到一条缓存记录

## 步骤6：配置真实天气API（可选）

如果需要获取真实天气数据：

- [ ] 访问 https://dev.qweather.com/ 注册账号
- [ ] 创建应用，选择"免费订阅"
- [ ] 获取 API Key
- [ ] 在 `cloudfunctions/getWeather/index.js` 中替换 API Key
- [ ] 重新部署云函数
- [ ] 测试真实天气数据

## 常见问题排查

### 问题1：天气数据不显示

检查项：
- [ ] 云函数是否已部署
- [ ] weather_cache 集合是否已创建
- [ ] 控制台是否有错误日志
- [ ] weatherData 是否有值（在控制台输入 `this.data.weatherData` 查看）

### 问题2：云函数调用失败

检查项：
- [ ] 云开发环境ID是否正确
- [ ] 云函数是否部署成功
- [ ] 网络是否正常
- [ ] 查看云函数日志（云开发控制台 → 云函数 → 日志）

### 问题3：缓存不生效

检查项：
- [ ] weather_cache 集合是否存在
- [ ] 集合权限是否正确
- [ ] 查看云函数日志，确认缓存操作

### 问题4：显示模拟数据

这是正常的！如果：
- [ ] 未配置和风天气API Key
- [ ] API调用失败
- [ ] 网络问题

云函数会自动返回模拟数据，确保小程序正常显示。
