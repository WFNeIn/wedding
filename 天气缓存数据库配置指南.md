# 天气缓存数据库配置指南

## 创建 weather_cache 集合

### 步骤1：打开云开发控制台

1. 在微信开发者工具中，点击顶部菜单栏的"云开发"按钮
2. 或者点击工具栏的"云开发"图标
3. 进入云开发控制台

### 步骤2：进入数据库管理

1. 在云开发控制台左侧菜单中，点击"数据库"
2. 进入数据库管理页面

### 步骤3：创建集合

1. 点击页面上的"添加集合"或"+"按钮
2. 在弹出的对话框中输入集合名称：`weather_cache`
3. 点击"确定"创建集合

### 步骤4：设置权限

1. 找到刚创建的 `weather_cache` 集合
2. 点击集合名称右侧的"权限设置"按钮
3. 选择权限模式：**仅创建者可读写**
4. 点击"确定"保存

## 集合结构说明

`weather_cache` 集合用于存储天气数据缓存，结构如下：

```javascript
{
  _id: "自动生成的ID",
  cacheKey: "weather_30.8263_114.8113",  // 缓存键（基于经纬度）
  date: "2024-01-01",                     // 日期（YYYY-MM-DD格式）
  weatherData: {                          // 天气数据
    weather: "晴",
    temp: "25",
    tempRange: "18~28",
    windDir: "东南风",
    windScale: "3级",
    humidity: "65",
    text: "晴朗",
    updateTime: "2024-01-01T12:00:00+08:00"
  },
  createTime: "服务器时间",               // 创建时间
  updateTime: "服务器时间"                // 更新时间
}
```

## 索引配置（可选）

为了提高查询性能，可以添加索引：

1. 在 `weather_cache` 集合页面，点击"索引管理"
2. 点击"添加索引"
3. 添加以下索引：

### 索引1：cacheKey + date（复合索引）
- 字段1：`cacheKey`，排序：升序
- 字段2：`date`，排序：降序
- 索引名称：`cacheKey_date`
- 说明：用于快速查询指定位置和日期的缓存

### 索引2：date（单字段索引）
- 字段：`date`
- 排序：升序
- 索引名称：`date`
- 说明：用于清理过期缓存

## 验证配置

创建完成后，可以在云开发控制台的数据库页面看到 `weather_cache` 集合。

## 测试云函数

1. 在云开发控制台，点击左侧菜单的"云函数"
2. 找到 `getWeather` 函数
3. 点击"测试"按钮
4. 输入测试参数：
```json
{
  "latitude": 30.826285,
  "longitude": 114.811298
}
```
5. 点击"运行测试"
6. 查看返回结果和日志

## 常见问题

### Q1: 集合创建失败
**A:** 检查云开发环境是否已开通，确保有足够的权限。

### Q2: 权限设置找不到
**A:** 在集合列表中，点击集合名称右侧的"..."菜单，选择"权限设置"。

### Q3: 云函数调用失败
**A:** 
1. 确保云函数已部署（右键云函数文件夹 → 上传并部署）
2. 检查云开发环境ID是否正确配置
3. 查看云函数日志，定位具体错误

### Q4: 缓存数据过多
**A:** 云函数会自动清理7天前的旧缓存，无需手动清理。
