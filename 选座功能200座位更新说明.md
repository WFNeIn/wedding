# 选座功能200座位更新说明

## 更新概述

选座功能已从40个座位升级到200个座位，支持20个桌子的宴会厅布局。

## 主要变更

### 1. 座位布局

**之前（40座位）**：
- 左侧：4排5列 = 20个座位
- 右侧：4排5列 = 20个座位
- 总计：40个座位

**现在（200座位）**：
- 左侧：10个桌子 × 10人/桌 = 100个座位
- 右侧：10个桌子 × 10人/桌 = 100个座位
- 总计：200个座位（20个桌子）
- 备用桌：第10桌（左右各1个，共20个座位）

### 2. 数据模型变更

#### 座位标识
```javascript
// 之前
{
  row: 1-4,      // 排号
  col: 1-5,      // 列号
  seatId: "left_1_1"
}

// 现在
{
  tableNum: 1-10,     // 桌号
  seatNum: 1-10,      // 座位号
  seatId: "left_1_1", // 左侧第1桌第1号座位
  isReserveTable: false  // 是否备用桌
}
```

#### 座位状态
- `available` - 空闲座位（可选择）
- `occupied` - 已占用（不可选择）
- `reserved` - 备用桌（不可选择，灰色显示）

### 3. 前端界面更新

#### 选座页面
- 按桌子分组显示
- 每个桌子显示10个座位（5×2网格布局）
- 桌号标签显示（1桌、2桌...10桌）
- 备用桌标注"(备用)"且显示灰色
- 点击备用桌提示"该桌为备用桌，暂不开放选座"

#### 管理员页面
- 统计面板增加"备用桌"数量显示
- 座位布局按桌子分组
- 备用桌座位显示灰色

## 兼容性说明

### 向后兼容
- 如果数据库中已有旧格式数据（row/col），需要手动清理或迁移
- 建议在正式使用前清空seats集合

### 数据迁移（如需要）
```javascript
// 如果有旧数据需要迁移，可以使用以下脚本
// 在云开发控制台的云函数中执行

const db = cloud.database();
const _ = db.command;

// 查询所有旧格式数据
const oldSeats = await db.collection('seats')
  .where({
    row: _.exists(true)
  })
  .get();

// 转换并更新
for (let seat of oldSeats.data) {
  // 旧格式无法直接映射到新格式
  // 建议直接删除旧数据，让用户重新选座
  await db.collection('seats').doc(seat._id).remove();
}
```

## 常见问题

### Q: 为什么要设置备用桌？
A: 备用桌用于应对临时增加的宾客，或者作为机动座位使用。

### Q: 备用桌可以手动分配吗？
A: 目前备用桌不开放在线选座，需要管理员线下安排。

### Q: 如何修改备用桌的桌号？
A: 在代码中搜索`tableNum === 10`，修改为其他桌号即可。

### Q: 可以设置多个备用桌吗？
A: 可以，修改验证逻辑，例如：`tableNum >= 9`表示9桌和10桌都是备用桌。

### Q: 座位数量可以继续调整吗？
A: 可以，修改`generateInitialSeats()`函数中的循环范围即可。
