# 婚礼现场抽奖大屏幕使用说明

## 功能说明

这是一个用于婚礼现场抽奖的大屏幕页面,可以在电脑浏览器中打开,投影到大屏幕上进行现场抽奖。

## 使用步骤

### 1. 打开大屏幕页面

在电脑浏览器中打开 `lottery-screen/index.html` 文件,或者将此文件部署到服务器后访问。

### 2. 选择奖项

点击页面上的奖项按钮:
- 一等奖 (1名)
- 二等奖 (2名)
- 三等奖 (3名)

### 3. 开始抽奖

1. 点击"开始抽奖"按钮,号码开始快速滚动
2. 再次点击"停止"按钮,号码停止滚动
3. 显示中奖号码
4. 系统会自动记录中奖名单

### 4. 继续抽奖

- 选择下一个奖项
- 重复步骤3
- 直到所有奖项抽完

## 功能特点

1. **防重复**: 已中奖的号码不会再次被抽中
2. **限额控制**: 每个奖项只能抽取指定数量
3. **中奖名单**: 实时显示所有中奖号码
4. **视觉效果**: 炫酷的滚动动画和中奖提示

## 小程序端配置

### 1. 创建云数据库集合

在微信开发者工具的云开发控制台中创建以下集合:

#### lottery_numbers (抽奖号码表)
```json
{
  "_id": "自动生成",
  "openid": "用户openid",
  "number": "001",
  "numberValue": 1,
  "createTime": "创建时间",
  "hasPrize": false
}
```

#### lottery_prizes (中奖记录表)
```json
{
  "_id": "自动生成",
  "winnerNumber": "001",
  "prizeName": "一等奖",
  "prizeLevel": 1,
  "drawTime": "抽奖时间"
}
```

### 2. 配置数据库权限

将两个集合的权限设置为:
- 所有用户可读
- 仅创建者可写(或所有用户可写,根据需求)

### 3. 创建login云函数(可选)

如果需要获取用户openid,需要创建login云函数:

```javascript
// cloudfunctions/login/index.js
const cloud = require('wx-server-sdk')
cloud.init()

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  return {
    openid: wxContext.OPENID,
    appid: wxContext.APPID,
    unionid: wxContext.UNIONID,
  }
}
```

## 抽奖流程

### 宾客端(小程序)
1. 宾客打开小程序,进入"我的号码"页面
2. 系统自动分配唯一号码(001-200)
3. 宾客记住自己的号码
4. 等待现场抽奖

### 现场端(大屏幕)
1. 主持人打开大屏幕抽奖页面
2. 选择要抽取的奖项
3. 点击开始,号码快速滚动
4. 点击停止,显示中奖号码
5. 宣布中奖号码
6. 继续抽取其他奖项

### 领奖流程
1. 中奖者打开小程序"我的号码"页面
2. 页面会显示中奖信息
3. 凭小程序页面到指定地点领奖

## 注意事项

1. **提前测试**: 婚礼前一天测试所有功能
2. **网络准备**: 确保现场有稳定的网络
3. **备用方案**: 准备纸质号码作为备用
4. **奖品准备**: 提前准备好所有奖品
5. **工作人员**: 安排专人负责核对号码和发放奖品

## 高级功能(可选)

如果需要将大屏幕抽奖结果同步到小程序,可以:

1. 将大屏幕页面改造为小程序页面
2. 使用云数据库实时同步中奖信息
3. 中奖者小程序自动收到通知

## 故障排除

### 问题1: 小程序无法分配号码
- 检查云开发环境是否正确配置
- 检查数据库集合是否创建
- 查看控制台错误信息

### 问题2: 大屏幕页面无法打开
- 确认文件路径正确
- 使用现代浏览器(Chrome/Edge)
- 检查浏览器控制台错误

### 问题3: 号码重复
- 大屏幕会自动检测重复
- 如果出现重复,点击重新抽取
